## Your First Conversion

Let's convert a PDF to Markdown, then convert that Markdown to a DOCX file.

<Steps>
  <Step title="Convert a file to Markdown">
    Upload any document (PDF, DOCX, Excel, etc.) to the `/upload` endpoint:

```bash
curl -X POST http://localhost:8000/upload \
  -F "files=@document.pdf" \
  -o output.md
```

The API returns a clean Markdown file. If you upload multiple files or an Excel workbook with multiple sheets, you'll get a ZIP archive instead.
  </Step>

  <Step title="Inspect the Markdown">
    Open `output.md` to see your converted content:

```markdown
# Document Title

Your content is now in clean, portable Markdown format with:
- Headings and paragraphs preserved
- Tables converted to Markdown tables
- Images referenced with standard syntax
- Lists and formatting maintained
```
  </Step>

  <Step title="Convert Markdown to DOCX">
    Now convert the Markdown back to a formatted DOCX file:

```bash
curl -X POST http://localhost:8000/convert \
  -F "file=@output.md" \
  -F "format=docx" \
  -F "use_default_template=true" \
  -o final.docx
```

You now have a professionally formatted DOCX file created from Markdown.
  </Step>

  <Step title="Open your document">
    Open `final.docx` in Microsoft Word or any compatible application. Your content is now in a polished, shareable format.
  </Step>
</Steps>

## Try Other Formats

The API supports multiple output formats:

<Tabs>
  <Tab title="PowerPoint">
    Convert Markdown to PPTX presentations:

```bash
curl -X POST http://localhost:8000/convert \
  -F "file=@slides.md" \
  -F "format=pptx" \
  -F "use_default_template=true" \
  -o presentation.pptx
```

Each level-1 heading (`#`) becomes a new slide.
  </Tab>

  <Tab title="HTML">
    Convert Markdown to standalone HTML:

```bash
curl -X POST http://localhost:8000/convert \
  -F "file=@document.md" \
  -F "format=html" \
  -o webpage.html
```

The output is a complete HTML file ready to publish.
  </Tab>
</Tabs>

## Using Custom Templates

Apply your own branding by uploading a custom DOCX or PPTX template:

```bash
curl -X POST http://localhost:8000/convert \
  -F "file=@document.md" \
  -F "format=docx" \
  -F "use_default_template=false" \
  -F "template=@my-template.docx" \
  -o branded.docx
```

<Note>
Learn more about creating and using custom templates in the [Custom Templates guide](/guides/custom-templates).
</Note>

## Check Template Availability

Before converting, you can check which default templates are available on the server:

```bash
curl http://localhost:8000/convert/templates
```

Response:

```json
{
  "default_templates": {
    "docx": true,
    "pptx": true
  }
}
```

## What's the Upload Limit?

By default, the API accepts files up to 50 MB. You can configure this via the `MAX_UPLOAD_MB` environment variable:

```bash
MAX_UPLOAD_MB=100
```

See the [Configuration](/deployment/configuration) page for all available settings.
