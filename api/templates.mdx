## Overview

The `/convert/templates` endpoint returns information about which default templates are available on the server for DOCX and PPTX conversions.

## Endpoint

```
GET /convert/templates
```

## Request

This endpoint does not require any parameters.

### Request Example

<CodeGroup>

```bash cURL
curl http://localhost:8000/convert/templates
```

```python Python
import requests

response = requests.get("http://localhost:8000/convert/templates")
data = response.json()
print(data)
```

```javascript JavaScript (fetch)
const response = await fetch('http://localhost:8000/convert/templates');
const data = await response.json();
console.log(data);
```

```javascript JavaScript (axios)
import axios from 'axios';

const response = await axios.get('http://localhost:8000/convert/templates');
console.log(response.data);
```

</CodeGroup>

## Response

### Response Body

<ResponseField name="default_templates" type="object">
  An object containing the availability status of default templates for each supported format
  
  <ResponseField name="default_templates.docx" type="boolean">
    Whether a default DOCX template exists on the server
  </ResponseField>
  
  <ResponseField name="default_templates.pptx" type="boolean">
    Whether a default PPTX template exists on the server
  </ResponseField>
</ResponseField>

### Response Example

```json
{
  "default_templates": {
    "docx": true,
    "pptx": true
  }
}
```

### When Templates Are Missing

If templates haven't been generated yet:

```json
{
  "default_templates": {
    "docx": false,
    "pptx": false
  }
}
```

## Status Codes

<ResponseField name="200" type="Success">
  Successfully retrieved template availability information
</ResponseField>

## Implementation Details

### Template Location

The endpoint checks for template files in the `TEMPLATES_DIR` directory (configured in `app/config.py`):

```python
TEMPLATES_DIR = BASE_DIR / "templates"
```

### Template File Names

The expected default template filenames are:
- `default.docx` for Word documents
- `default.pptx` for PowerPoint presentations

### Template Generation

Default templates are automatically generated on server startup by the `ensure_default_templates()` function (app/main.py:14), which:
1. Checks if template files exist
2. If missing, generates them using Pandoc's built-in defaults
3. Saves them to the templates directory

## Use Cases

### Client-Side Feature Detection

Use this endpoint to determine whether to show template-related options in your UI:

```javascript
const response = await fetch('http://localhost:8000/convert/templates');
const { default_templates } = await response.json();

if (default_templates.docx) {
  // Show "Use default template" checkbox for DOCX
}
```

### Health Check with Template Validation

Combine with the health endpoint to ensure the server is fully configured:

```python
import requests

# Check API health
health = requests.get("http://localhost:8000/health").json()

# Check template availability
templates = requests.get("http://localhost:8000/convert/templates").json()

if health["status"] == "ok" and all(templates["default_templates"].values()):
    print("API is fully operational")
else:
    print("API is running but templates may be missing")
```

### Diagnostic Information

Use in troubleshooting to verify template setup:

```bash
# Quick check of template availability
curl -s http://localhost:8000/convert/templates | jq
```

## Notes

- This endpoint is read-only and does not modify server state
- Template availability is checked in real-time by verifying file existence
- Missing templates don't prevent the API from running, but `use_default_template=true` won't have any effect
- HTML conversions don't use templates, so only DOCX and PPTX are reported
